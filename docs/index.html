GET /stats/count

Tags: Statistic, Symptom, Occurrence

Compute the number of occurrences that occurred for all the symptoms of the current user. The default unit to count the occurrences is per 'days' but it can also be counted per 'hours', 'months' and 'years'. The list of symptoms can be filtered via the 'symptoms' parameter that receives a list of 'symptom_id'. The occurrences can be filtered by using the 'start' and 'end' date parameters.
start

Start date of the occurrences to count. Any occurrences that were created before this date will be ignored.

 query string (date-time)
end

End date of the occurrences to count. Any occurrences that were created after this date will be ignored.

 query string (date-time)
unit

The 'unit' parameter must take one of the following value: 'hours', 'days', 'months', 'years'. The default value is per 'days'. This parameter determine how to count occurrences.

 query string , x ∈ { hours , days (default) , months , years }
symptoms

An array of symptoms' id to include in the result. Only those symptoms will be included in the result. If this parameter is not present, all the symptoms will be included.

 query integer[]

Uses default content-types: application/json

200 OK

An array of SymptomsCounts

SymptomsCounts
422 Unprocessable Entity

Invalid parameter, the end date is before the start date.

A factor describe more accuratly an occurrence of a symptom. For example it can describe the intensity of pain or the precise location of a pain.
